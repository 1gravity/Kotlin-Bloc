The CoroutineRunner is tied to a bloc either to run initializer, thunk or reducer coroutines. It's using one of the three bloc CoroutineScopes -> it's tied to the bloc's lifecycle.

Unfortunately the CoroutineRunner needs to be public because it's exposed by public functions (InitializerContext, ThunkContext, ReducerContext etc.).
