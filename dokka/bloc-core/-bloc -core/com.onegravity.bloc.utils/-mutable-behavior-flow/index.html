A MutableSharedFlow with

  • an initial value

  • no duplicates

  • value property

We use this "enhanced" MutableSharedFlow over MutableStateFlow because the latter conflates values. See: https://stackoverflow.com/questions/73583072/kotlin-flow-with-initial-value-replay-1-no-duplicates-no-conflation
